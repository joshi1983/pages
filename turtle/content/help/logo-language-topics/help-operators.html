The Logo programming language supports two general types of operators.

  1. Operators that compare and evaluate to either true or false. These are symbols like >, <.
  2. Operators that evaluate to numbers. These are arithmatic operators like +, -, *, /.

Arithmatic Binary Operators

Unary Operators

A unary operator is an operator that takes 1 input. For example, the negative sign. A unary operator appears before its input in a Logo program similar to how a command appears before its input.

Comparison Operators

Comparison operators are useful for if, ifelse, and while-loop commands.

Precedence

Precedence determines the order in which operators are processed.

If you ever want more control over the order of operations in evaluating your expressions, use curved brackets ( ). The brackets are always evaluated before anything else.

The higher the precedence, the earlier the operator is evaluated. At equal precedence, operators are evaluated left to right.

All possible command and procedure parameters are evaluated before invoking the command or procedure. Let's look at some examples to make this clearer.

  • print 3 > 2 prints true. It is the same as print (3 > 2). It isn't the same order as (print 3) > 2. A little off topic but (print 3) > 2 would also cause an error since print never returns a valid input to the > operator.

  • if sum 3 2 > 0 [ print "hi ] won't run. It is the same as if sum 3 (2 > 0) [ print "hi ]. It can't run because > returns either true or false which is not a valid input for the sum command. An extra reason for it to not run is that sum returns a number and the if-statement's condition must be boolean(either true or false).

    If you run into a situation like this and hate the order of operation, use brackets to take more control over the order. Consider something like: if (sum 3 2) > 0 [ print "hi ] which will run and print hi because (sum 3 2) will evaluate to 5 and 5 > 0 evaluates to true.

  • print 3 + 2 prints 5. It is the same as print (3 + 2).
