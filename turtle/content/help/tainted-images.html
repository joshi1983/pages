An image can be added to a drawing using the image or imageAlpha commands. An HTTP or HTTPS protocol URL for the source parameter of those commands often adds a "tainted" image to the drawing. In other words, the image becomes troublesome for some features of WebLogo to work with.

If you want more technical details, this "tainted" image is more of a "tainted" HTML5 canvas caused by drawing an unsafe or tainted image onto the canvas. The security issue that causes canvases to become tainted is explained in detail at https://developer.mozilla.org/en-US/docs/Web/HTML/CORS_enabled_image

Some features won't work on drawings with tainted images.

  • Drawing -> Download feature from working for JPG, PNG, WebP formats due to an underlying browser security feature.
  • Since you can't download raster images from drawings containing tainted images, you also can't export animations that contain tainted images.
  • Under Set -> Set Time, the Live Redraw will stop working

Workarounds

You can add an image as an asset and reference it using a url like local://myimage.png assuming you have a PNG image asset named myimage.png. This is usually the best workaround since it keeps your code small. Click File -> Assets to access the asset manager.

You can work around this by using base-64 data URL's. The drawback is your URL will become very long. It'll quite surely be too long see within the code editor unless you scroll.
