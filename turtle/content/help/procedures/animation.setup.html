animation.setup

The animation.setup procedure returns a property list defining information about the animation as a whole. animation.setup should never call animation.time or try to draw anything.

Properties

  • duration is the number of seconds of duration
  • thumbnailTime is the number of seconds to the best still frame to depict the animation. This time is the same frame initially shown in File -> Load Example

Don't

The animation.setup is not to be used for anything except defining properties for the overall animation listed above. Keep that clear by avoiding the following mistakes.

  • Don't use a case for property names other than the ones specified above. For example, don't use DURATION instead of duration because the property names in a property list are case-sensitive.
  • Don't add parameters to the animation.setup procedure since it should never have any.
  • Don't call animation.duration within animation.setup. The result of animation.duration is the same as the duration property of animation.setup's procedure. Calling animation.duration is both asking for an endless cycle and serves no sensible purpose.
  • Don't call animation.time within animation.setup. animation.setup should behave the same way irrespective of time. Calling animation.time within animation.setup is a bit like asking what happened before the existence of time.
  • Don't try to draw anything in animation.setup. Don't call commands like forward in animation.setup. animation.setup is for indicating settings for your animation.

Example

The following indicates the animation is 10 seconds long.

to animation.setup localmake "minutes 0 localmake "seconds 10 ; Edit this if you want to change the length of your animation. output createPList2 [ ["duration :minutes * 60 + :seconds] ] end

Note that you can quickly add code for animation.setup by clicking Edit -> Set up Animation from the Code Editor.

See also animation.snapshotstyle. animation.snapshotstyle is another special procedure related to animation.
