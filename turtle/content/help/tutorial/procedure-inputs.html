When you call penUp or penDown, no inputs are needed. When you use the forward command, you give it a number to tell it how far to move forward. You can define these extra inputs in your procedures too.

Here is a procedure to draw a square using a size input:

to square :size
  repeat 4 [
    forward :size
    right 90
  ]
end

square 100

These inputs or parameters are declared using a few rules:

  • A procedure input must be on the same line as the procedure's name
  • The input must start with a colon
  • The input name must follow all the rules of a variable name. This includes:
    • The character after the colon must not be a digit or a period even though digits and periods can be used anywhere later in the name
    • Can contain letters, underscores anywhere
