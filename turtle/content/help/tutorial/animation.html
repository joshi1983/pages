Making drawings can be exciting but you may want to animate your drawing. Maybe you want to draw something different as time changes.

Step 1

The quickest first step to converting your drawing into an animation is to click Edit -> Set up Animation in your code editor's menu. It will prompt you for how long you want your animation to last. Clicking "Add animation setup" will add the necessary procedures to your code.

animation.setup

The animation.setup procedure returns the number of seconds that your animation will last. This procedure should have a very simple implementation that basically returns a number. animation.setup should always return the same number and therefore it should never call animation.time.

animation.snapshotstyle

The animation.snapshotstyle procedure returns a property list that expresses zoom and camera position. The animation.snapshotstyle properties are used when rendering frames and snapshots from your animation. You will often want to call animation.time within your animation.snapshotstyle procedure to change your zoom and camera position based on time.

A frame may be made out of more than one snapshot if you export an animation using simulated motion blur.

Step 2

Use animation.time. animation.time returns the number of seconds into your animation. Your drawing will animate simply by changing how it looks based on the value of animation.time.

Example with rotating line

right animation.time * 36
forward 100
; The above 2 lines represent an animation that rotates a line around the home position ; The full 360 degree rotation will take 10 seconds.

Without the animation procedures before, the animation will default to 10 seconds long.

Example Including the Procedures

Defining the animation.snapshot and animation.setup procedures will make the properties of your animation more clear. Below is the example after clicking Edit -> Set Up Animation as described in Step 1.

right animation.time * 36
forward 100
; The above 2 lines represent an animation that rotates a line around the home position ; The full 360 degree rotation will take 10 seconds.
; The below procedures express that the animation is 10 seconds long and that only default snapshot properties are to be used.

to animation.snapshotstyle
  localmake "result createPList
  output :result
end

to animation.setup
  localmake "minutes 0
  localmake "seconds 10
  ; Edit this if you want to change the length of your animation.
  output :minutes * 60 + :seconds
end
