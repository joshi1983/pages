The animation exporter is available for all animated drawings by following these steps:

  1. Click Drawing -> Download.
  2. Click Animation button.

An animated drawing or animation defined by aspects of the code in the code editor. Having any of the following in your code will cause the Animation button to be visible in the Drawing -> Download feature.

  • Calls animation.time, animation.timeRatio, animation.clampedTimeRatio
  • Defines a procedure called animation.setup

File Menu

An animation can be downloaded as an Animated GIF or a Frame Sequence.

  • An animated GIF can be useful for short, very low-resolution animations. An animated GIF should generally be under 10 seconds long, use less than 100 frames, and be under 600 by 600 pixels.
  • A frame sequence can be useful for high resolution or longer videos. A frame sequence enables you to process the video more with other software such as ffmpeg to combine the frames into MP4 files.

The prefix is where you select the start to your downloaded file name. For an Animated GIF, the prefix is the single file name before the trailing .gif. For a frame sequence, the prefix is the part before the frame index digits and file extension.

Animated GIF Options

Repeat Count is the number of times the animation should repeat after opening it. 1 or Infinite are what most people would choose. Infinite means the GIF animation will repeat as long as it remains open.

Frame Sequence Options

Start index shows only when you're exporting a Frame Sequence. It is the first frame index to be downloaded and defaults to 0.

End index

End index shows only when you'r exporting a Frame Sequence. The end index is the last index to be downloaded and defaults to -1. An end index of -1 means you want to download to the very last frame of the animation.

The End index input will turn yellow if its value is not valid with respect to the current start index. If the end index is positive but less than start index, no frames will be selected which is very likely not what you want and the input turns yellow to warn you.

The File format for each frame can be JPG, PNG, or WEBP.

Lossless Quality

The Lossless Quality checkbox lets you choose between frames that are

  • Lossless and large file sizes
  • Lossy but small file sizes

By default, lossless is checked and usually that is preferred. A compressed MP4 video will lose some quality but most people don't want to sacrifice any quality until encoding that final version.

Resolution Menu

You can select from various resolutions.

For animated GIF's, a low resolution is generally recommended.

If you want to share a video on youtube, exporting 4K Ultra HD or higher is generally the best because some people may watch your videos on 4K or 8K screens. Your high resolution video will downscale very well to lower resolution screens.

Timing Menu

The Frame rate is how many frames you want per second. A low frame rate under 24 is often used with animated GIF's. 24, 25, and 30 are generally preferred if you want to make a video for social media. 30fps is generally the best for facebook and youtube.

The Motion blur snapshots per frame lets you select how many snapshots should be blended together for each frame. The trade-offs are essentially this. If you use more snapshots per frame, you get a higher quality, smoother looking blur effect. The downside of a high number of snapshots per frame is that your download takes longer to complete. A high snapshot per frame number is usually most important when aspects of the video very rapidly move or change.
