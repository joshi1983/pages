The Code Editor's Edit -> Format Code menu item replaces all indentation with tabs and breaks lines to a consistent and clear format.

There are many reasons to carefully format your code.

  • Consistent indentation improves readability of the code.
    • Where a loop begins and ends becomes clearer when the whole loop isn't on a single line.
    • The order of instructions is clearer when they're more strictly up and down instead of having many on the same line.
  • Short code on each line makes it easier to benefit from other features of WebLogo.
    • Breakpoints can be added in more places when there are more lines to add them to.
    • When your program runs, the progress indicator shows which line is currently executing. It is more informative to know that a line calling 1 command is executing than a line containing 10 commands.

For the above reasons, maintaining consistent and clear formatting is important in any code you write.

The Format Code feature is useful in the following situations.

  • You may copy Logo code examples from another websites. Logo programmers sometimes put too many command calls on the same line and manually formatting this can be tedious.
  • You manually formatted your code but want any small inconsistencies fixed.
